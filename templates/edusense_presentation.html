Deep Learning Model

EduSenseTransformer:
A Novel Architecture

Transformer Encoder + KAN + CORAL for ordinal emotion classification — an original contribution.

🎬

Input

30 frames per clip (10s video at 3fps)

👁️

AffectNet-ViT

Frozen ViT · 768-dim emotion-aware embeddings

🧠

Transformer Encoder

4 layers · 12 heads · CLS token · Pos. embeddings

〰️

KAN Layer

B-spline basis · 768→256 · Learnable activations

🎯

CORAL Heads ×4

Binary ordinal classifier per emotion

✅

Predictions

Boredom · Confusion · Frustration · Engagement

Transformer over LSTM

Sees all 30 frames simultaneously using attention — LSTM forgets early frames. This was the biggest single accuracy improvement in the project.

KAN over MLP

B-spline basis functions capture non-linear emotional patterns that standard MLPs miss. Learnable activation functions per neuron connection.

CORAL over Softmax

Ordinal classification preserves the natural ordering of engagement levels (high → medium → low). Binary classifiers per ordinal threshold.

7-Model Ensemble

Seven diverse Transformers with different random seeds. Averaged probabilities with per-emotion threshold tuning gave 81.1% final accuracy.

Detection Pipeline

Real-Time Emotion
Detection Pipeline

From raw webcam frame to actionable dissatisfaction score in milliseconds.

1

Frame Capture

Webcam captures a BGR frame every 2 seconds. Resized to 320×240 for processing efficiency.

2

CLAHE Preprocessing

Contrast Limited Adaptive Histogram Equalization in LAB color space. Handles dim rooms and bright windows robustly.

3

FER Detection

FER library outputs 7 raw emotion probabilities. Normalized to sum=1. Requires confidence > 0.3 to proceed.

4

Baseline Calibration

First 30 frames build each student's personal neutral/happy baseline. Adjusts for their unique resting face expression.

5

State Formula Mapping

D'Mello & Graesser (2012) formulas map raw emotions to learning states. Confusion uses a multiplicative synergy of fear × surprise.

6

Temporal Smoothing

7-frame exponential sliding window. Requires 3 consecutive threshold crossings before triggering. Eliminates false alarms.

Challenges & Solutions

Every Obstacle Made
the System Stronger

Six real engineering problems encountered and solved during development.

⚠ Severe Class Imbalance
93.4% of DAISEE clips are "engaged." Naive training achieves 95% accuracy by predicting engaged for everything — completely useless as a dissatisfaction detector.
Focal CORAL loss + asymmetric class weights (up to ×4.5) + per-emotion threshold tuning + dedicated specialist models for failing classes.
⚠ CORAL Loss Implementation Bug
Early implementation had incorrect ordinal label construction and reversed prediction logic. The model appeared to learn (79% accuracy) but predicted nothing — misleading metrics hid the failure completely.
Complete rewrite of loss function, prediction decoding, and per-epoch TP/FP/FN monitoring per emotion. Never trust accuracy alone on imbalanced data.
⚠ ChromaDB Python 3.13 Incompatibility
ChromaDB had no Python 3.13 support at development time — blocking the entire RAG system from running on modern Python environments.
Replaced with FAISS (faiss-cpu). IndexFlatIP provides exact cosine similarity with zero version conflicts. Persisted with pickle.
⚠ DeepFace GPU Dependency
DeepFace required a dedicated GPU for real-time inference, making the web app unusable on standard student laptops without hardware.
Replaced with FER library (CPU-friendly). CLAHE preprocessing compensates for reduced model capacity. Runs on any machine.
⚠ False Trigger Prevention
Momentary expressions — yawning, looking away briefly, adjusting glasses — caused false dissatisfaction signals and unnecessary notebook generation.
7-frame exponential sliding window + 3 consecutive threshold crossings required + personal baseline calibration for each student's resting face.
⚠ Engagement AUC Near Random
Engagement showed AUC ≈ 0.50 when used as a dissatisfaction trigger — adding pure noise rather than signal to the detection system.
Excluded engagement entirely from the trigger logic. System now triggers only on boredom, confusion, or frustration — much more reliable.

How It Works — Part 1

From Your Face
to a Trigger Signal

What actually happens every 2 seconds while you watch a lecture.

📷
01

Webcam Captures Frame

Every 2 seconds the browser sends a Base64-encoded webcam frame to the Flask backend via /api/analyze-frame.

MediaRecorder API → Base64 → Flask
🌓
02

CLAHE Preprocessing

OpenCV converts the frame to LAB color space and applies Contrast Limited Adaptive Histogram Equalization — making detection robust to any lighting condition.

BGR → LAB → CLAHE → BGR
😐
03

FER — 7 Raw Emotions

The FER library detects the largest face and returns 7 raw emotion scores: happy, neutral, fear, surprise, angry, disgust, sad. Scores are normalized to sum = 1.

FER → {happy, neutral, fear, surprise, angry, disgust, sad}
🎯
04

Personal Baseline Calibration

The first 30 frames build a personal neutral/happy baseline for this specific student. A naturally serious face won't falsely trigger boredom.

baseline_neutral = mean(neutral[0:30])
🧮
05

D'Mello Formula Mapping

Raw FER scores are mapped to 4 learning states using formulas from D'Mello & Graesser (2012). Confusion uses a multiplicative synergy: fear × surprise — both must be elevated simultaneously.

confusion = (fear × surprise × 6.0) + fear×0.25 + surprise×0.20
⚡
06

Trigger Decision

A 7-frame exponential sliding window smooths predictions. If boredom, confusion, or frustration stays above threshold for 3 consecutive checks — the RAG pipeline fires. 60-second cooldown prevents spam.

trigger_count ≥ 3 AND now − last_trigger > 60s
How It Works — Part 2

From a Trigger
to a Notebook in Seconds

What happens the moment EduSense decides a student is struggling.

🚨
01

Trigger Fired

Flask detects 3 consecutive dissatisfied frames. It records the exact YouTube video timestamp and the detected emotions (boredom / confusion / frustration).

struggle_moment → {video_time, detected, transcript}
🎤
02

Whisper Grabs Context

Whisper has already transcribed the full lecture audio via yt-dlp. The system retrieves the last 60 seconds of speech at the exact trigger timestamp.

get_last_n_seconds(video_time, n=60) → transcript
🔍
03

FAISS Retrieval

The transcript is embedded with SentenceTransformer (all-MiniLM-L6-v2) into a 384-dim vector. FAISS IndexFlatIP searches the textbook knowledge base and returns the top-5 most relevant chunks.

embed(transcript) → FAISS.search(k=5) → chunks[]
🤖
04

Claude API Generation

A structured prompt containing the transcript, top-4 textbook chunks, and detected emotion is sent to claude-sonnet-4. Claude returns a strict JSON object with title, explanation, analogy, key points, code examples, and exercises.

prompt(transcript + chunks + emotion) → JSON → .ipynb
📓
05

Notebook Delivered

The JSON is assembled into a full .ipynb Jupyter notebook with markdown and code cells. It's saved to disk and served via /api/download/<filename> — ready to download instantly, timestamped to the exact moment of confusion.

JSON → to_ipynb() → /api/download/20250101_120000_topic.ipynb
