RLS — Students

Students read/write only their own sessions, struggle moments, emotion history, and notebooks. Other students' data is invisible at the database level.

RLS — Teachers

Teachers read all sessions and struggle moments where teacher_id = their ID. They cannot access other teachers' class data.

Auto-Profile Trigger

A PostgreSQL trigger automatically creates a profiles row when a user signs up via Supabase Auth. No orphaned users, no race conditions.

Service Role vs Anon Key

Flask uses the service_role key to write emotion data in real time (bypasses RLS). The anon key is used only in the frontend for safe public operations.

Authentication & User System New

Two Roles, One Platform —
JWT Authentication

Students and teachers sign up separately. Each gets a completely different dashboard. Every API call is authenticated via JWT.

POST /api/auth/signup

Creates a Supabase Auth user with role, full_name, department, student_id or faculty_id in user metadata. A PostgreSQL trigger instantly creates the profiles row.

sb.auth.sign_up({ email, password, options: { data: { role, full_name, … } } })

POST /api/auth/login

Returns a JWT access_token + full profile row. Frontend stores token in localStorage (with in-memory fallback for Edge browser). Every request sends Authorization: Bearer <token>.

sb.auth.sign_in_with_password({ email, password }) → JWT

JWT Middleware — get_current_user()

Every protected Flask route validates the JWT via sb.auth.get_user(token). Returns the Supabase user object or None. Returns 401 if token missing or invalid.

Authorization: Bearer <token> → sb.auth.get_user(token)

Edge Browser Fallback

A safe storage wrapper falls back to in-memory storage when Edge's tracking prevention blocks localStorage. student_id is also sent in the request body as a JWT-free fallback for start-session.

store.get / store.set / store.del — memory fallback on catch

Deep Learning Model

EduSenseTransformer —
A Novel Architecture

Transformer Encoder + KAN + CORAL for ordinal emotion classification — an original research contribution.

🎬

Input

30 frames per clip (10s at 3fps)

👁️

AffectNet-ViT

Frozen ViT · 768-dim emotion embeddings

🧠

Transformer Encoder

4 layers · 12 heads · CLS token

〰️

KAN Layer

B-spline basis · 768→256

🎯

CORAL Heads ×4

Binary ordinal classifier per emotion

✅

Predictions

Boredom · Confusion · Frustration · Engagement

Transformer over LSTM

Sees all 30 frames simultaneously using self-attention — LSTM forgets early frames. Largest single accuracy jump in the project.

KAN over MLP

B-spline basis functions capture non-linear emotional patterns that standard MLPs miss. Learnable activation per connection.

CORAL over Softmax

Ordinal classification preserves the natural ordering of engagement levels. Binary classifiers per threshold avoid rank inconsistency.

7-Model Ensemble

Seven diverse Transformers with different seeds. Averaged probabilities with per-emotion threshold tuning. Final: 81.1%.

How It Works — Part 1

From Your Face
to a Trigger Signal

What actually happens every 2 seconds while you watch a lecture.

📷
01

Webcam Captures Frame

Every 2 seconds the browser draws the webcam feed onto a canvas and encodes it as Base64 JPEG. Sent via POST to /api/analyze-frame.

canvas.toDataURL('image/jpeg', 0.7) → POST /api/analyze-frame
🌓
02

CLAHE Preprocessing

OpenCV converts to LAB color space and applies Contrast Limited Adaptive Histogram Equalization — robust to dim rooms and bright windows.

BGR → LAB → CLAHE(clipLimit=2.5, tileGrid=4×4) → BGR
😐
03

FER — 7 Raw Emotions

FER library detects the largest face and returns 7 raw scores: happy, neutral, fear, surprise, angry, disgust, sad. Normalized to sum=1. Rejected if total confidence < 0.3.

FER → {happy, neutral, fear, surprise, angry, disgust, sad} / total
🎯
04

Personal Baseline Calibration

First 30 frames build a personal neutral/happy baseline for this specific student. A naturally serious face won't falsely trigger boredom.

baseline_neutral = mean(neutral[0:30]) · neutral_adj = n − baseline × 0.6
🧮
05

D'Mello Formula Mapping

Raw FER scores mapped to 4 learning states. Confusion uses multiplicative synergy — fear AND surprise must both be elevated simultaneously to trigger.

confusion = (fear × surprise × 6.0) + fear×0.25 + surprise×0.20
⚡
06

Trigger + Supabase Persist

7-frame exponential sliding window smooths predictions. Trigger fires at count ≥ 2 with a 60-second cooldown. Struggle moment written immediately to Supabase with video timestamp, emotions, and transcript.

trigger_count ≥ 2 AND now − last_trigger > 60s → sb.table('struggle_moments').insert(…)
How It Works — Part 2

From a Trigger
to a Notebook in Seconds

What happens the moment EduSense decides a student is struggling.

Knowledge Pipeline
📄

PDF Upload

Professor uploads lecture textbooks. PyMuPDF extracts and chunks paragraphs longer than 50 characters.

🔢

Embedding

SentenceTransformer (all-MiniLM-L6-v2) converts each chunk to a 384-dim vector, normalized for cosine similarity.

🗄️

FAISS Index

IndexFlatIP stores embeddings on disk. No ChromaDB — works on Python 3.13 with zero version conflicts.

🎤

Whisper Transcription

Lecture audio downloaded via yt-dlp in a background thread. Last 60 seconds used as retrieval context at trigger time.

🤖

Claude API Generation

Transcript + top-5 textbook chunks + detected emotion → claude-sonnet-4 → strict JSON → .ipynb served instantly.

💻 Code Examples

🏋️ Practice Exercises

Questions with hints + collapsible solution cells

📖 Further Reading

Pointer to relevant textbook chapter for deeper study

Challenges & Solutions

Every Obstacle Made
the System Stronger

Key technical challenges encountered and the engineering decisions that solved them.

⚠ FER False Positives
Naturally serious faces triggered false boredom alerts constantly. Threshold-only detection was unreliable across different students.
Personal baseline calibration from first 30 frames adjusts each student's threshold individually. Eliminated false alarms by ~70%.
⚠ DAISEE Class Imbalance
Frustration had only 222 training clips (4%). Standard ensemble collapsed to predicting the majority class — 0% Frustration recall.
Dedicated specialist models for Frustration and Engagement. Custom per-class loss weighting. Separate threshold tuning per emotion.
⚠ Python 3.13 Compatibility
ChromaDB and many vector store libraries all failed on Python 3.13 — unusable on the target machine.
Switched to FAISS IndexFlatIP with manual numpy persistence. Zero external vector DB dependency. Works on any Python 3.11+.
⚠ Edge Browser localStorage Blocking
Edge's tracking prevention silently blocked localStorage, so auth tokens were never saved — every API call returned "Unauthorized".
Built a safe storage wrapper that falls back to in-memory storage. Also added student_id in the request body as a JWT-free fallback.
⚠ Session Already Active
If a student closed the browser without ending a session, Flask's in-memory session dict stayed active=True, blocking all new sessions.
Auto-reset logic in start-session: if active=True, force-reset before creating the new session. No user action required.
⚠ Supabase Profile Missing
Email confirmation enabled during testing caused Supabase auth trigger to miss creating the profiles row for early test accounts.
Disabled email confirmation. Added ON CONFLICT DO NOTHING to the trigger. Manual SQL insert for existing test accounts.
